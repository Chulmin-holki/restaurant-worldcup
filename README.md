# 뭐먹지 월드컵

캐치테이블 음식점 상세 화면 캡처와 링크를 넣어 카드형 토너먼트로 고르는 모바일 우선 웹앱입니다. 별도 설치 없이 GitHub Pages에서 실행되며, 휴대폰 홈 화면에 추가하면 앱처럼 사용할 수 있습니다.

## 들어 있는 기능

- `[기념일] 뭐먹지 월드컵`처럼 주제 입력
- 음식점마다 파일 선택창을 한 번 열어 캡처 1~2장을 복수 선택
- 선택 순서대로 첫 번째는 기본 정보, 두 번째는 AI 요약으로 처리
- 공유 문구 전체를 붙여넣어도 그 안의 `https://...` 주소만 자동 추출
- 캡처에서 이름·위치·별점·리뷰·요약 자동 인식 후 수정
- 캡처 상단 대표 사진을 토너먼트 카드와 우승 화면에 활용
- AI 요약 캡처는 선택 사항이며, 음식점별 다시 인식과 대표 사진 위치 조정 지원
- OCR 진행 단계, 불확실한 항목, 중복 링크 안내
- 화면 전체가 아닌 고정된 기본 정보·AI 요약 영역을 확대 인식하고 깨진 `AI가 요약했어요` 제목은 결과에서 제외
- 서로 다른 화면 비율을 위한 기본 정보 영역 2회 인식, 흑백 대비 보정, 줄 단위 이름 선택 및 별점·리뷰 독립 추출
- 영문·한글 혼합 상호명 한 줄 보존, 수도권 지역 확장, 리뷰 수 천 단위 쉼표 표시
- `한 남`, `한남역`, `한남동`처럼 공백이나 주변 표현이 포함돼도 `한남`으로 정규화
- 주제와 입력 링크 임시 저장(보안상 사진 파일은 새로고침 후 다시 선택 필요)
- 첫 화면에서 저장된 주제·링크·선택한 캡처를 확인 후 전체 초기화
- 후보 이름·위치 확인 화면
- 확인 화면을 공유하면 받는 사람이 바로 후보 확인 화면부터 시작
- 위로 스와이프해 토너먼트 시작
- 위 카드는 왼쪽, 아래 카드는 오른쪽으로 스와이프해 선택
- 마우스 드래그와 선택 버튼도 지원
- 랜덤 매칭과 부전승 처리
- 부전승 후보는 다음 라운드 첫 매치에 배치
- 모든 라운드 시작 전에 `1라운드 · 8강`, `2라운드 · 4강`, `결승 · 2강` 안내 화면
- 우승 시 팡팡 효과와 상세 정보 표시
- PWA 지원: 홈 화면에 추가하고 오프라인에서 앱 화면 실행 가능

## GitHub Pages에 올리는 가장 쉬운 방법

1. 이 폴더의 압축을 풉니다.
2. GitHub에 로그인한 뒤 오른쪽 위 `+` 버튼을 누르고 `New repository`를 선택합니다.
3. `Repository name`에 `food-worldcup`을 입력합니다.
4. 공개 사이트로 쓸 예정이면 `Public`을 선택합니다.
5. `Create repository`를 누릅니다.
6. 새 저장소 화면에서 `uploading an existing file`을 누릅니다. 이 문구가 안 보이면 `Add file` → `Upload files`를 누릅니다.
7. 압축을 푼 폴더 **안의 파일 전체**를 업로드 영역으로 끌어놓습니다. 폴더 자체가 한 겹 더 들어가지 않게 주의합니다.
8. 아래 `Commit changes`를 누릅니다.
9. 저장소 위쪽 `Settings` → 왼쪽 `Pages`로 이동합니다.
10. `Build and deployment`의 `Source`를 `Deploy from a branch`로 선택합니다.
11. Branch는 `main`, 폴더는 `/(root)`를 선택하고 `Save`를 누릅니다.
12. 같은 화면에 나타나는 `Visit site`를 누르거나 `https://아이디.github.io/food-worldcup/` 주소를 엽니다. 첫 배포에는 최대 10분 정도 걸릴 수 있습니다.

## 휴대폰에서 앱처럼 쓰기

- iPhone Safari: 공유 버튼 → `홈 화면에 추가`
- Android Chrome: 오른쪽 위 메뉴 → `홈 화면에 추가` 또는 `앱 설치`

## 캐치테이블 자동 가져오기에 관한 중요한 설명

GitHub Pages는 정적 웹사이트라서, 다른 서비스가 브라우저의 외부 읽기를 차단하면 캐치테이블 페이지 내용을 직접 가져올 수 없습니다. 이 프로젝트는 먼저 자동 가져오기를 시도하고, 실패하면 입력 화면으로 자연스럽게 전환합니다. 링크만으로 항상 자동 가져오게 하려면 선택 사항인 `catchtable-proxy-worker.js`를 별도 프록시로 배포하고 `config.js`의 `importProxy`에 그 주소를 넣어야 합니다.

프록시를 연결해도 캐치테이블의 페이지 구조나 정책이 바뀌면 자동 가져오기가 다시 제한될 수 있습니다. 따라서 직접 입력 대체 화면은 삭제하지 않는 편이 안전합니다.

## 파일 설명

- `index.html`: 앱 시작 파일
- `styles.css`: 전체 디자인과 애니메이션
- `app.js`: 가져오기, 공유, 스와이프, 토너먼트 로직
- `config.js`: 선택 사항인 가져오기 프록시 주소
- `manifest.webmanifest`, `service-worker.js`, `icon.svg`: 홈 화면 설치와 오프라인 실행
- `catchtable-proxy-worker.js`: 선택 사항인 자동 가져오기 보조 코드
- `.nojekyll`: GitHub Pages가 파일을 그대로 배포하도록 하는 설정

## 직접 수정할 때

- 메인 색상: `styles.css` 맨 위의 `--accent`
- 앱 이름: `index.html`과 `manifest.webmanifest`
- 자동 가져오기 프록시: `config.js`

공유 링크에는 주제와 후보 정보가 주소의 `#worldcup=...` 부분에 들어갑니다. 그래서 데이터베이스 없이도 받는 사람이 첫 화면을 건너뛰고 후보 확인 화면부터 시작할 수 있습니다.
